1. Reply Guy на Threads

Язык: RU | Цель: подписчики Threads → трафик в TG @danielspe_chanel
Пошаговая схема 
ЭТАП 1: МОНИТОРИНГ (ежедневно, 10 мин)

  Инструмент: Playwright MCP (browser_navigate + browser_snapshot)
  Действие: открыть threads.net → For You → фильтр по нише AI/Product/Tech
  Критерий: пост набрал 50+ лайков за < 2 часов = "залетевший"
  Результат: список 3-5 постов для комментирования

ЭТАП 2: АНАЛИЗ ПОСТА (Claude Code, 2 мин на пост)

  Инструмент: Playwright → browser_snapshot → Claude анализирует текст
  Действие: извлечь ключевую тему, тезис, спорный момент
  Результат: черновик комментария (200-400 символов)

ЭТАП 3: НАПИСАНИЕ КОММЕНТАРИЯ (формула)

  ЯКОРЬ:  "Кстати, про [конкретный пункт из поста] -"
  ЦЕННОСТЬ: "У нас в InsurTech похожая задача, мы решили через [кейс с цифрами]"
  КРЮЧОК: "Подробнее с кодом описал у себя в профиле"
  Длина: 150-300 символов, разговорный тон, на русском

ЭТАП 4: ПУБЛИКАЦИЯ (Playwright, 1 мин)

  Инструмент: Playwright MCP → browser_click → browser_type
  Задержка: 3-5 сек между действиями (human-like)
  Проверка: browser_snapshot → убедиться что коммент опубликован

ЭТАП 5: РЕЗУЛЬТАТ

  Люди видят коммент → заходят в профиль → видят посты на RU
  В bio: "Автоматизирую бизнес AI-агентами. Кейсы → @danielspe_chanel"
  Конверсия: ~2-5% читателей комментария → подписчик TG

2. 70/30 Reply Machine на X

Язык: RU (посты/треды) + язык оригинала (replies) | Цель: подписчики X → TG
Пошаговая схема 
УТРЕННИЙ ЦИКЛ (8:00-9:00, 60 мин)

  1. МОНИТОРИНГ ТРЕНДОВ (15 мин)
     Инструмент: Talkwalker Alerts → email с трендами за ночь
     Инструмент: hnrss.org RSS → посты > 50 очков по AI/automation
     Инструмент: Playwright → X поиск "AI agent" OR "AI automation" → Top
     Результат: 5-10 трендовых постов для ответов

  2. ГЕНЕРАЦИЯ REPLIES (20 мин, Claude Code)
     Инструмент: Claude Opus анализирует каждый пост
     Формула reply:
       [Согласие/уточнение конкретного пункта]
       [Свой опыт с цифрами: "Мы сэкономили X$ делая Y"]
       [Вопрос автору ИЛИ крючок: "описал кейс в треде"]
     Результат: 10 черновиков replies (150-250 символов)

  3. ПУБЛИКАЦИЯ REPLIES (15 мин, Playwright MCP)
     Инструмент: Playwright → последовательно, 1 reply за раз
     Задержка: 30-90 сек между replies (рандом)
     Проверка: screenshot после каждого reply

  4. СВОЙ КОНТЕНТ (10 мин)
     Инструмент: X API (curl) - публикация 1-2 постов на русском
     Формат: hook (250 символов) + CTA → "Больше → @danielspe_chanel"
     2 раза/неделю: тред (7 твитов) вместо одиночного поста

3. First Responder (мониторинг трендов)

Все платформы | Цель: быть в первых 3-5 комментариях на трендовых постах

     Частота: as-it-happens (реальное время)
     Доставка: email → logvas7@gmail.com

     https://hnrss.org/newest?q=AI+agent&points=50

     Добавить в Feedly

4. Воронка VC.ru → Telegram

Язык: RU | Главный бесплатный драйвер подписчиков TG
Пошаговая схема 
ПОДГОТОВКА СТАТЬИ ДЛЯ VC.ru

  1. Статья (2000+ слов, русский)
     Инструмент: Claude Code → перевод/адаптация pillar article
     Вставить 2-3 нативные ссылки на TG:
       После инсайта: "Подробнее разбираю в канале @danielspe_chanel"
       В середине: "Исходный код выложил в канале @danielspe_chanel"
       В конце: CTA "Больше кейсов AI-автоматизации → t.me/danielspe_chanel"
     НЕ делать: спамные CTA, >3 ссылок (минусы от читателей)

  2. Публикация (Playwright MCP, ~5K токенов)
     Подзаголовок с интригой
     Обложка: 1920x1080, тёмная с цифрами
     Категория: AI или Стартапы (больше охват)

  3. Разгон (первые 2 часа)
     Threads (RU): "Написал подробный кейс на VC.ru → [ссылка]"
     X (RU): тред-тизер с ключевыми цифрами + ссылка
     TG-канал: "Новая статья на VC → [ссылка]"
     Если > 10 плюсов в первый час → алгоритм буст → "Популярное"

  4. Результат
     "Популярное" VC.ru = 5K-50K просмотров
     Конверсия в TG: 0.5-2% от просмотров
     Ожидание: 25-200 подписчиков TG с одной статьи

5. Публикация одной статьи (3 дня)

1 статья → 14+ площадок в 4 языках | Подробнее: Пайплайн
Пошаговая схема 
ДЕНЬ 1: EN Original (30 мин, ~5K токенов)

  Claude Code: адаптировать markdown → формат каждой платформы
  API (параллельно, 5 мин):
    Dev.to EN     → curl POST (canonical → Medium)
    Hashnode EN   → curl GraphQL
    Bluesky EN    → atproto SDK
    Mastodon EN   → curl POST
  Playwright (последовательно, 25 мин):
    Medium EN     → canonical URL source
    LinkedIn EN   → пост + ссылка
    Substack EN   → newsletter

ДЕНЬ 2: RU + Соцсети (25 мин, ~4K токенов)

  Claude Code: перевод → RU + генерация соцсетей постов
  API (5 мин):
    Telegram RU   → MCP send_message
    X тред RU     → X API (7 твитов)
    Threads RU    → Meta API (пост)
  Playwright (последовательно, 20 мин):
    VC.ru RU      → Editor.js
    Хабр RU       → markdown
    Medium RU     → перевод

ДЕНЬ 3: ES/PT (15 мин, ~3K токенов)

  API (параллельно, 5 мин):
    Dev.to ES/PT  → curl POST
    TabNews PT    → curl POST
  Playwright (10 мин):
    Medium ES/PT  → Playwright

ИТОГО: ~12K токенов за 3 дня (вместо ~50K+ при чистом Playwright)

6. Универсальная формула комментария

ЯКОРЬ:   "Отличный пункт про [конкретная часть поста]."
ЦЕННОСТЬ: "В моём опыте, [свой инсайт/данные/кейс]."
КРЮЧОК:  "Описал похожий кейс [у себя / в треде / в TG]"

Примеры по платформам:

Threads (RU):
"Точно! Мы тоже автоматизировали отчётность AI-агентами - за 2 месяца заменили 3 человека. Самое сложное было не технология, а доверие команды. Подробнее у меня в профиле"

X (RU):
"У нас похожий кейс в InsurTech - 17 AI-скиллов вместо финотдела. Экономия $15K/мес. Главный инсайт: начинать не с самого сложного, а с того что больнее всего. Тред с деталями →"

LinkedIn (EN):
"Great point about AI replacing processes, not people. We went through exactly this - automated financial reporting with AI agents, saved $15K/month. The key was starting with the most painful task, not the easiest one. Happy to share the full breakdown."

HN/Reddit (EN):
"Built something similar for InsurTech reporting. 17 Claude Code skills replaced manual reconciliation across 200+ partners. Main gotcha: the API rate limits were the real bottleneck, not the AI quality."

Формула X тредов (7 твитов)

Твит 1: HOOK (80% успеха - bold statement + tension)
        "Я заменил финотдел AI-агентом за 2 месяца. Вот что я узнал →"

Твит 2-3: CONTEXT (зачем это важно)
        "200+ партнёров, 3 человека на отчётности, каждый месяц одни и те же ошибки..."

Твит 4-6: VALUE (1 идея = 1 твит, < 250 символов)
        "Шаг 1: Автоматизировал сбор данных из БД (экономия: 40 часов/мес)"

Твит 7: CTA
        "Если хочешь повторить - полный гайд с кодом у меня в канале.
         RT если полезно"
